# [3주차] 농기계 통신 네트워크 (CAN & ISOBUS) 데이터 분석

## 1. 실습 개요
본 실습에서는 트랙터 및 작업기 간 통신의 핵심인 **CAN Bus (ISO 11898)**와 **J1939/ISOBUS** 프로토콜의 원리를 이해합니다. 실제 기계에서 수집된 로우(Raw) 데이터를 직접 파싱하여 인간이 읽을 수 있는 물리값(엔진 속도, 차량 속도 등)으로 변환하는 프로토콜 디코더를 구현합니다.

---

## 2. 주요 실습 내용
- 29비트 CAN ID 비트 연산을 통한 **Priority, PGN, Source Address** 추출
- **Little-Endian** 바이트 정렬 방식의 이해 및 구현
- PGN별 스케일(Scale) 및 오프셋(Offset) 적용 실습

---

## 3. 핵심 개념: J1939 ID 구조
- **Priority (3 bits)**: 메시지 우선순위 (0이 최고순위)
- **PGN (18 bits)**: 메시지 그룹 번호 (Parameter Group Number)
- **Source Address (8 bits)**: 메시지를 송신하는 ECU 주소

---

## 4. 실습 코드 실행 안내

### [Step 1] 파이썬을 이용한 CAN 디코딩 실습
제공된 `step1_can_decoder.py` 파일을 실행하여 샘플 로그의 해석 결과를 확인합니다.

```bash
# 코드 실행
python step1_can_decoder.py
```

### [Step 2] 데이터 해석 과제
코드 내의 `logs` 리스트에 새로운 데이터를 추가하여 결과를 도출해 보십시오.
1. `0CF00400` ID의 PGN이 `61444`임을 비트 연산으로 증명할 수 있는가?
2. 냉각수 온도가 168도(과열 상황)로 출력되는 원인을 오프셋(-40) 관점에서 설명할 수 있는가?

---

## 5. 실습 결과 해석 가이드
- **디코딩 성공**: 전문적인 CAN 분석기 없이도 코드 한 줄로 트랙터의 상태를 모니터링할 수 있음
- **확장성**: PGN 테이블을 확장하면 유압 압력, PTO 속도 등 기계의 모든 정보를 통합 관리할 수 있음
